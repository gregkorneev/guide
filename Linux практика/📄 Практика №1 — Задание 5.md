## **Работа с файлами и каталогами в Alt Linux**

---

## **🎯 Цель работы**

  

Освоить базовые команды Linux для работы с файловой системой: создание каталогов и файлов, копирование, перемещение, удаление и проверка чувствительности к регистру.

---

## **🔹 1. Определение текущего каталога**

```
pwd
```

---

## **🔹 2. Просмотр содержимого каталога**

```
ls
```

Расширенный вариант:

```
ls -l
```

---

## **🔹 3. Создание каталогов**

  
Создадим структуру каталогов:

```
mkdir practice
cd practice
mkdir dir1 dir2 dir3
```

Проверка:

```
ls
```

---

## **🔹 4. Создание файлов**

```
touch file1.txt
touch file2.txt
```

Проверка:

```
ls
```

---

## **🔹 5. Просмотр содержимого файла**

```
cat file1.txt
```

---

## **🔹 6. Копирование файлов**

```
cp file1.txt dir1/
```

Проверка:

```
ls dir1
```

---

## **🔹 7. Перемещение (или переименование) файлов**

```
mv file2.txt dir2/
```

Переименование:

```
mv file1.txt file1_renamed.txt
```

---

## **🔹 8. Удаление файлов и каталогов**

Удаление файла:

```
rm file1_renamed.txt
```

Удаление каталога:

```
rm -r dir3
```

---

## **🔹 9. Проверка чувствительности к регистру**

  

Создадим два файла:

```
touch test.txt
touch Test.txt
```

Проверка:

```
ls
```

📌 Вывод:

Linux различает test.txt и Test.txt — это разные файлы.

---

## **🔹 10. Использование справки**

```
man ls
```

Выход из справки:

```
q
```

---

# **🍎 macOS**

Все команды выполняются **аналогично** в терминале macOS при подключении по SSH:

```
ssh username@192.168.1.100
```

---

# **🪟 Windows**

## **PowerShell:**

```
ssh username@192.168.1.100
```

---

# **📌 Результат выполнения**

  

В ходе работы:

- Созданы каталоги и файлы
    
- Выполнены операции копирования и перемещения
    
- Проверена чувствительность Linux к регистру
    
- Освоены команды работы с файловой системой
    

---

# **⚠️ Возможные ошибки**

|**Ошибка**|**Причина**|**Решение**|
|---|---|---|
|No such file|Файл не существует|Проверить имя|
|Permission denied|Нет прав|Использовать sudo|
|Directory not empty|Папка не пустая|Использовать rm -r|
