# **📄 Практика №1 — Задание 1**

  

## **Подключение к Alt Linux по SSH**

---
## **🔹 1. Определение IP-адреса виртуальной машины**
ste

В терминале Alt Linux выполнить:

```
ip a
```

Найти строку вида:

```
inet 192.168.1.100/24
```

📌 IP-адрес: 192.168.1.100

---

## **🔹 2. Проверка наличия SSH-сервера**

```
systemctl status sshd
```

### **Если SSH не установлен:**

```
su -
apt-get update
apt-get install openssh-server
```

---

## **🔹 3. Запуск SSH-сервера**

```
systemctl start sshd
systemctl enable sshd
```

Проверка:

```
systemctl status sshd
```

Ожидаемый результат:

```
active (running)
```

---

# **🍎 Подключение с macOS**

  

## **🔹 4. Подключение по SSH**

  

В терминале macOS:

```
ssh username@192.168.1.100
```

📌 Где:

- username — пользователь Alt Linux
    
- 192.168.1.100 — IP виртуальной машины
    

---

## **🔹 5. Первый вход**


При первом подключении:

```
Are you sure you want to continue connecting (yes/no)?
```

Ввести:

```
yes
```

Затем ввести пароль.

---

## **🔹 6. Проверка подключения**

```
whoami
pwd
```

---

# **🪟 Подключение с Windows**


## **🔹 Вариант 1 — через PowerShell**

  

Открыть PowerShell:

```
ssh username@192.168.1.100
```

Дальше аналогично macOS.

---

## **🔹 Вариант 2 — через PuTTY**

1. Открыть **PuTTY**
    
2. В поле **Host Name** ввести:
    

```
192.168.1.100
```

2.   
    
3. Порт: 22
    
4. Нажать **Open**
    
5. Ввести логин и пароль

# **⚠️ Возможные ошибки**

|**Ошибка**|**Причина**|**Решение**|
|---|---|---|
|Connection refused|SSH не запущен|systemctl start sshd|
|Permission denied|Неверный пароль|Проверить ввод|
|No route to host|Нет сети|Проверить настройки VM|
|Timeout|Неправильный IP|Проверить ip a|
